# 練習播放與完成回饋 UX 設計（已取代）

> 本文件描述的播放方案已被 `2026-07-17-practice-feedback-simplification-design.md` 取代；目前產品不提供播放，按鍵解說只使用題目推薦解法。

## 目標

改善練習頁的四個使用體驗問題：

1. Level 4 播放時，使用者不必手動捲動才能看到播放內容。
2. 播放時，完成後的程式碼內容要清楚可見。
3. 題目成功後保留目前的編輯器，不再以回饋畫面取代編輯器。
4. 題目完成後提供只包含本題實際使用按鍵的 Vim 按鍵解說。

這次只調整前端顯示與互動，不改變評分、題目判定、Attempt 結構、IndexedDB 或 Supabase 同步行為。

## 設計決策

### 1. 播放區位置與速度

Level 4 解鎖後，播放元件會顯示在練習編輯器區塊附近，而不是只放在頁面底部的提示清單中。播放區包含：

- 播放按鈕。
- 逐顆高亮的 Vim 按鍵序列。
- 唯讀的「完成後內容」預覽。

使用者按下播放按鈕時，播放區先以 `scrollIntoView({ behavior: "smooth", block: "center" })` 捲至視線中央，再開始播放。播放中的目前按鍵會持續留在編輯器附近，並以較慢的節奏逐顆顯示；預設每顆按鍵間隔為 600ms，保留可測試的 `stepDelayMs` 覆寫能力。

播放只展示完成後內容，不展示操作前內容，也不在內容上加入額外的 diff 標記。完成後內容取自目前題目的 `expectedContent`，播放不會修改使用者正在練習的 CodeMirror 編輯器。

播放完成後沿用既有行為：發出重設事件，清除示範狀態並要求使用者親自完成；動畫本身不建立 Attempt。

### 2. 成功後保留編輯器

完成成功後，`PracticePage` 不再以 `v-if="!feedback"` 隱藏整個練習工作區。編輯器、狀態列與完成後內容會留在頁面上，單題回饋改為接續顯示在工作區下方。

完成後的編輯器保留使用者剛完成的內容與游標，讓使用者能直接對照「你的操作」與畫面結果。完成後不再觸發自動評估，也不增加第二筆 Attempt；既有 `feedback !== null` 的提交保護仍是唯一提交閘門。

跳過或重新開始等練習控制項在保存結果期間仍遵守既有 disabled 行為。這次不新增編輯器資料保存欄位，也不改變題目切換流程。

### 3. 完成後的按鍵解說

新增 `VimKeyGuide`，由 `ExerciseFeedback` 在完成回饋中呈現，使用 `<details>` 且預設收合。練習進行中不顯示。

解說內容只從本次 Attempt 的 `NormalizedAction[]` 產生：

- 只列出使用者實際操作中出現的 Vim command、模式切換、Undo 與搜尋按鍵。
- 插入的業務文字不當成 Vim 按鍵解說，避免把任意程式碼內容列入速查表。
- 同一按鍵或組合只顯示一次，依首次出現順序排列。
- 已知按鍵顯示簡短繁體中文說明；未收錄的有效按鍵仍保留按鍵標籤，但不影響完成與評分。

按鍵解說是回饋層的展示資料，不寫入資料庫。`AttemptFeedback` 會攜帶供回饋元件使用的標準化操作資料或已整理的解說項目，既有 `userSequence` 與 Attempt payload 維持不變。

## 元件與資料流

```text
PracticePage
├─ VimEditor（練習中與完成後保留）
├─ EditorPlayback（Level 4 時顯示在編輯器附近）
├─ ProgressiveHintPanel（只負責提示解鎖）
└─ ExerciseFeedback
   └─ VimKeyGuide（完成後、預設收合、只列實際使用按鍵）
```

`ProgressiveHintPanel` 保留提示順序與 `highestLevelChanged`，不再負責把播放元件固定在提示清單內。`PracticePage` 根據最高提示層級把 Level 4 的播放資料渲染到編輯器附近，並接收播放完成後的既有 reset flow。

## 測試策略

### Component tests

- `EditorPlayback`：播放按鈕會先呼叫平滑捲動、以 600ms 預設間隔逐顆高亮、完成後發出 `playbackComplete`，且只顯示完成後內容。
- `ProgressiveHintPanel`：仍逐層解鎖，Level 4 不會在提示清單內重複渲染播放元件。
- `VimKeyGuide`：預設收合，只渲染輸入操作中實際出現的按鍵；重複按鍵去重並保留首次順序。
- `ExerciseFeedback`：完成回饋包含按鍵解說區，但未完成或沒有操作時不渲染不存在的按鍵說明。

### Service tests

- 標準化操作可產生已使用按鍵與繁體中文說明。
- 插入文字不會被誤列為 Vim 按鍵。
- 未知有效按鍵保留標籤，不造成錯誤或改變評分。

### End-to-end tests

- Level 4 播放時，播放區與高亮按鍵出現在編輯器附近，按下播放後頁面執行平滑捲動，並以較慢速度完成播放。
- 播放完成後不建立 Attempt，編輯器仍可見。
- 題目成功後，編輯器仍可見且保留完成後內容；回饋顯示在其下方。
- 完成回饋的按鍵解說預設收合，展開後只看到本題實際使用過的按鍵。

## 非目標

- 不建立第二個可操作的 Vim 編輯器。
- 不把播放動畫當成使用者操作，不建立成功 Attempt。
- 不修改速度、準確、熟練計算或資料庫 schema。
- 不在練習中顯示完整按鍵速查表。
- 不新增未被本次需求要求的遊戲化或學習功能。
