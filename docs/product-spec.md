# Product Specification

## 1. 產品願景

建立一個類似 Epop 學習節奏的 Vim 練習網站，讓新手到進階使用者透過反覆操作，在真實程式碼情境中建立 Vim 肌肉記憶。

核心循環：

```text
選擇需求
→ 完成操作題
→ 得到速度、準確、熟練回饋
→ 系統記錄弱點
→ 之後再次複習
```

## 2. 目標使用者

### 2.1 從零開始

適用：

- 完全沒使用過 Vim。
- 不理解 Normal Mode 與 Insert Mode。
- 需要逐步教學與操作示範。

學習節奏：

```text
概念說明
→ 操作動畫
→ 指定按鍵
→ 引導題
→ 自主題
```

速度只供參考，準確優先。

### 2.2 記憶複習

適用：

- 曾學過 Vim，但常忘記指令。
- 知道 `ciw`、`di"`、`f`、`t`，卻不會自然使用。
- 希望大量反覆練習。

學習節奏：

```text
選擇題量
→ 自動複習或指定主題
→ 錯題再出現
→ 延長複習間隔
```

熟練與準確優先。

### 2.3 效率進階

適用：

- 已可用 Vim 日常編輯。
- 習慣大量 `h`、`l`、`x`，希望學更短操作。
- 想練習 Text Object、Operator + Motion、搜尋跳轉。

學習節奏：

```text
完成真實任務
→ 分析操作路徑
→ 顯示推薦解法
→ 使用推薦解法再做一次
```

準確是必要條件，速度是主要回饋。

## 3. 首頁流程

首頁顯示三張模式卡片：

### 從零開始

說明：

> 第一次使用 Vim？從模式切換與基礎移動開始。

按鈕：`開始學習`

### 記憶複習

說明：

> 學過但容易忘記？透過反覆出題建立肌肉記憶。

按鈕：`開始複習`

### 效率進階

說明：

> 已經會 Vim？學習更少按鍵、更有效率的操作方式。

按鈕：`挑戰效率`

登入且已有資料時，另外顯示：

- 繼續上次練習
- 今日待複習題數
- 建議加強技能

## 4. 訪客與登入

### 訪客

- 可瀏覽課程。
- 可進入三種模式。
- 可完成練習。
- 答題與熟練資料存於 IndexedDB。
- 重新整理後資料仍存在。

### 登入

- 使用 Google OAuth。
- 登入後將未同步訪客資料合併至帳號。
- 合併不能重複建立 Attempt。
- 合併失敗時保留本機資料。
- 登入後可跨裝置同步。

## 5. 題量

記憶複習與效率進階開始前可選：

| 題量 | 使用情境 |
|---:|---|
| 5 | 快速複習，約 3–5 分鐘 |
| 10 | 標準練習，約 8–12 分鐘 |
| 20 | 完整訓練，約 15–25 分鐘 |

預設 10 題。

從零開始模式依小節推進，每個小節建議：

- 1 個概念教學
- 1–2 個引導操作
- 2–4 個自主挑戰

## 6. 記憶複習入口

### 6.1 今日自動複習

選題比例：

- 70% 到期與錯題
- 20% 弱項補強
- 10% 已熟悉抽查

優先級：

1. 上次未完成。
2. 上次跳過。
3. 已超過到期時間。
4. 使用 Level 3 或 4 提示。
5. 準確偏低。
6. 按鍵效率偏低。

「今日待複習 N 題」在首頁、學習進度頁與今日複習頁必須顯示同一個數字——三者都來自同一份系統記住的到期複習排程，不是各頁各自估算。

### 6.2 指定主題

使用者可選一個或多個主題：

- 模式切換
- 基礎移動
- 單字移動
- 行內跳轉
- 刪除與修改
- 複製貼上
- 全文搜尋
- 文字物件
- Visual Mode
- 綜合操作

指定主題練習不會清除其他到期題。

## 7. 課程地圖

所有單元一開始即可進入。

| 單元 | 核心指令 | 題數 |
|---|---|---:|
| 模式切換與基本編輯 | `i a I A o O Esc` | 10 |
| 基礎游標移動 | `h j k l` | 8 |
| 單字與行內快速移動 | `w W b B e 0 ^ $` | 10 |
| 刪除與修改 | `x X d dd D dw de d$ c cc C cw ce` | 12 |
| 複製、貼上、復原與重複 | `y yy yw p P u Ctrl-r .` | 10 |
| 行內搜尋與精準跳轉 | `f F t T ; ,` | 10 |
| 全文搜尋與程式碼定位 | `/ ? n N * #` | 10 |
| 文字物件 | `iw aw i" a" i' a' i( a( i[ a[ i{ a{` | 14 |
| Visual Mode 與範圍操作 | `v V Ctrl-v o d c y > <` | 8 |
| Vim 組合思維與效率 | Count、Operator、Motion、Text Object 組合 | 8 |
| 合計 |  | 100 |

Visual Block 若 CodeMirror Vim Keymap 的相容性不足，可在 MVP 標記為「實驗性」，但不得因此移除一般 `v` 與 `V`。

## 8. 題型

每個技能可有：

1. `tutorial`：概念與動畫示範。
2. `guided`：明確要求使用指定操作。
3. `challenge`：只顯示目標。
4. `review`：用不同程式碼情境重複技能。

每日複習只使用 `challenge` 與 `review`。

## 9. 漸進提示

每題可提供 0–4 層提示，實際層級由題庫決定：

1. 解釋操作方向。
2. 提示指令類型。
3. 顯示部分按鍵。
4. 提供更完整的操作方向。

提示依層級數值由小到大解鎖，只在題庫實際存在的層級中前進；缺少的層級不會阻塞後續提示。系統不提供播放動畫，提示內容不會代替使用者完成題目。

## 10. 題目完成條件

完成條件可組合：

- 最終內容正確。
- 內容保持不變。
- 游標在精確位置。
- 游標在指定範圍。
- 完成時位於指定 Vim Mode。

內容正確但仍在 Insert Mode 時，可顯示：

> 內容已正確，請回到 Normal Mode。

### 10.1 練習編輯器互動

- 可編輯題目建立完成後，只在初次掛載時自動取得焦點。
- 新題與重新開始的題目使用題目 `initialCursor`；恢復未完成題目使用已保存的 `currentCursor`。
- 唯讀編輯器不得自動取得焦點；使用者主動移到其他控制項後，編輯器不得持續搶回焦點。
- 編輯器下方顯示目前 Vim Mode、從目前 attempt `startedAt` 計算的已練習時間，以及「重新開始本題」控制項。
- 已練習時間使用壁鐘差值並顯示為 `mm:ss`；評分仍只使用既有 duration 輸入與公式，不得作為新的計分輸入。
- 重新開始會捨棄尚未完成的 attempt，建立新的 `clientAttemptId` 與 `startedAt`，並還原初始內容、初始游標、Normal Mode、零按鍵、零操作、零提示與零 `resetCount`。
- 重新開始不得自動建立成功或失敗 Attempt。
- 使用者輸入有效 Vim 操作後，系統即時重新評估完成條件；不提供額外的「檢查答案」步驟。
- 題目首次在使用者互動後符合所有完成條件時，自動保存一次成功 Attempt 並顯示單題回饋；載入初始狀態時不得自動完成；此時 session 尚未前進。
- 單題回饋提供「再試一次」與「下一題」；前者以新 attempt 重做同一題，後者才使 session 前進一次。
- 單題完成或保存中（含載入下一題）期間，編輯器與其鍵盤事件 handler 一律鎖定為唯讀，不得再產生或更新 attempt draft。
- 游標完成條件為 `exact` 或 `range` 時，在編輯器中以黃色、透明、細邊框標示目標位置；`ignore` 不顯示目標框。
- 編輯器產生的有效 Vim command、插入文字、模式切換、Undo、Reset 與搜尋會保存為標準化操作序列，並在單題回饋中顯示實際操作。
- 完成回饋的「本題按鍵解說」直接顯示推薦 solution 的 `explanation`，不得由前端固定按鍵字典產生說明。

## 11. 單題回饋

完成後依序顯示：

1. 是否完成。
2. 準確。
3. 速度。
4. 熟練變化。
5. 使用者操作。
6. 推薦操作。
7. 改善原因。
8. 本題按鍵解說（推薦 solution 的 `explanation` 逐字顯示；explanation 為空白時不顯示此區塊）。
9. 再試一次／下一題。

操作序列必須顯示使用者實際完成題目時的標準化操作；若操作有效但不在題庫解法中，仍顯示該操作並標示為未收錄的有效操作。

本題按鍵解說必須使用題目作者定義的推薦解法 `explanation`，不得由前端固定按鍵字典或使用者實際輸入的操作序列產生描述。

回饋頁同時提供三項指標的簡短定義與可展開的計算方式，讓使用者知道分數代表什麼，而不需要閱讀內部程式碼。

範例：

```text
完成！

準確：100 — 一次完成
速度：78 — 流暢
熟練：2 → 3 — 熟悉

你的操作：
wwciwcustomerName<Esc>

推薦操作：
ciwcustomerName<Esc>

起始游標已位於變數名稱內，因此不需要先按 ww。
```

## 12. 三項指標

### 12.1 速度

由：

- 按鍵效率 60%
- 時間效率 40%

組成。

模式寬限：

- beginner：目標時間 × 2.0
- memory_review：目標時間 × 1.3
- efficiency：目標時間 × 1.0

顯示等級：

| 分數 | 等級 |
|---:|---|
| 90–100 | 極快 |
| 75–89 | 流暢 |
| 50–74 | 普通 |
| 1–49 | 可再精簡 |
| 0 | 未完成 |

### 12.2 準確

從 100 分開始扣除：

| 行為 | 扣分 |
|---|---:|
| 一般誤操作 | 5 |
| Undo | 3 |
| Reset | 12 |
| Hint Level 1 | 3 |
| Hint Level 2 | 8 |
| Hint Level 3 | 15 |
| Hint Level 4 | 30 |
| 跳過／未完成 | 設為 0 |

低效率不重複扣準確。

`resetCount` 扣分規則仍是計分公式的一部分，但重新開始／再試一次一律建立 `resetCount` 為 0 的全新 attempt，目前沒有任何使用者流程會產生非零 `resetCount`；單題回饋不得因此顯示「重新開始會扣分」的說明文字。

介面定義：準確表示是否一次到位；Undo 與提示會扣分，單純按鍵較多不會重複扣準確分。

### 12.3 熟練

顯示 0–5：

| 等級 | 名稱 |
|---:|---|
| 0 | 未學習 |
| 1 | 不熟 |
| 2 | 練習中 |
| 3 | 熟悉 |
| 4 | 熟練 |
| 5 | 已掌握 |

熟練是跨多次、不同題目、不同時間的長期指標，不能因一次答對直接升到高等級。

介面定義：熟練是 0–5 的長期程度，不是單題分數；速度則由按鍵精簡度 60% 與完成時間 40% 組成，並套用學習模式的時間寬限。

## 13. 程式碼情境

題庫比例：

- C# 60%
- TypeScript／JavaScript 20%
- 其他 20%

其他包含：

- JSON
- HTML
- CSS
- SQL
- Markdown

程式碼不得考驗複雜語法或商業邏輯；目標是 Vim 操作。

## 14. 主要頁面

- `/`：首頁與模式選擇
- `/courses`：十個課程單元
- `/courses/:unitSlug`：單元內容
- `/practice/setup`：模式、題量、主題選擇
- `/practice/:sessionId`：練習
- `/practice/:sessionId/result`：整輪摘要
- `/review`：今日複習與指定主題
- `/progress`：熟練度與紀錄
- `/settings`：顯示與編輯器設定
- `/auth/callback`：Google OAuth 回呼
- Catch-all：404

## 15. 設定

MVP 設定：

- 顯示名稱
- 編輯器字體大小
- 顯示行號
- 顯示最近按鍵
- 開啟或關閉音效
- 預設題量
- 登出

## 16. MVP 不包含

- 排行榜、好友、聯賽
- XP、等級、徽章、愛心、金幣
- 付費功能
- 題庫管理後台
- AI 出題
- 外掛
- 自訂後端
- 完整手機練習
- 巨集、Registers、Marks、Jump List、Buffer、Window、跨檔案操作

## 17. 產品完成標準

MVP 必須讓使用者：

- 不登入即可開始。
- 自由選三種模式。
- 自由選課程單元。
- 選 5、10、20 題。
- 在 CodeMirror 使用 Vim。
- 看到速度、準確、熟練。
- 離線時不遺失已完成題目。
- Google 登入後同步訪客紀錄。
- 得到 70/20/10 的今日複習。
- 可指定主題練習。
